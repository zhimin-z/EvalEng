No infrastructure exists for defining or executing hypothesis tests (t-tests, permutation tests, chi-square tests, etc.). The bootstrap function (get_bootstrap_accuracy_std) computes standard deviation for visualization purposes, not hypothesis testing. No statistical test configuration options are available in eval specifications or the CLI.

The evaluation framework focuses on single-eval metrics rather than comparative analyses requiring multiple testing corrections.
   - Documentation Quality: None

No fields exist for hypothesis statements, predicted outcomes, or research questions. The eval workflow (evals/eval.py) implements only sample evaluation and metric aggregation, with no support for declaring hypotheses or comparing observed results against predictions. Documentation (docs/build-eval.md, docs/custom-eval.md) focuses entirely on dataset preparation and metric computation.

5. Analysis Plan Immutability
   - Support: 0
   - Evidence: No mechanisms exist to lock or make analysis plans immutable after specification. The eval registry system (evals/registry.py) loads YAML configurations dynamically at runtime with no versioning, hashing, or integrity checking to prevent modifications. The RunSpec class (evals/base.py) generates a run_id timestamp but does not store or validate pre-specified analysis procedures. YAML eval definitions can be freely modified without any system-enforced immutability. No cryptographic signing, version control integration, or audit logging exists to detect post-hoc changes to analysis specifications.
