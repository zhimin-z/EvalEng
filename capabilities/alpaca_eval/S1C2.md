## Executive Summary

The AlpacaEval harness implements **solid schema validation and consistency mechanisms** with explicit type checking, structured validation of required fields, and range/bound constraints. The system detects distribution biases and includes backward compatibility for schema evolution. However, formal training-serving consistency checks and comprehensive drift detection mechanisms are partially implemented.

   - Valid set: {1.0, 1.5, 2.0} (plus NaN for errors)
   - Constraint: Exactly 3 scalar values in valid range [1, 2]

3. **Output Pair Uniqueness**
   - Constraint: `output_1 != output_2` (enforced via derangement)
   - Function: `random_derangement()` (utils.py:63-83)
   - Ensures: No trivial pairs with identical outputs
   - Algorithm: Rejection sampling or full derangement computation

4. **Instruction Uniqueness**
   - Constraint: One pair per unique instruction (optional)
   - Parameter: `is_unique_instructions` (pairwise_evaluator.py:70, 136)
   - Default: True (enforced during pair creation)

   - Code:

   ```python
           all_outputs.groupby(keys_to_sample_output_2)
           .first()
           .reset_index()
       )
   ```

5. **Parser Output Cardinality**
   - Constraint: Parser must return list with one element per input example
   - Batch validation: Length must match input batch size
   - Enforced implicitly by vectorized operations

### 8. **Drift Detection** ~ **PARTIAL (65%)**

**Evidence: PARTIAL** - Bias detection implemented; statistical drift detection limited

**Drift Detection Mechanisms:**

1. **Length Bias Detection**
   - Function: `get_length_biases()` (analyze.py:326-359)
   - Metric: Probability of preferring longer outputs
   - Calculation: Correlation between output length and preference
   - Detects: Systematic preference for verbose outputs
   - Returns: Dictionary with `probability_prefer_longer` and `percentage_longer`
   - Test example (test_analyze.py:66-80):
   ```python
   def test_get_length_biases(analyzer):
       df = pd.DataFrame({
           "instruction": ["A", "B", "C"],
           "output_1": ["long output", "short", "much longer output" * 30],
           "output_2": ["short", "somewhat short", "shorter output"],
           "preference": [2, 1, 1],
       })
       result = analyzer.get_length_biases(df)
       assert result["probability_prefer_longer"] == pytest.approx(1.0)
   ```

2. **List Format Bias Detection**

   ```python
   bullet_point_pattern = r"(?m)^\s*[\*\•\-]\s*[^\w]*\w+"
   ```

   - Test example (test_analyze.py:83-96):

   ```python
   def test_get_list_biases(analyzer):
   ```
