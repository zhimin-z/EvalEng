High-Coverage Phases
Step 4 - SCORE Quantus excels at converting raw explanations into quantified metrics. The library provides 35+ production-ready metrics across 6 categories (Faithfulness, Robustness, Localisation, Complexity, Randomisation, Axiomatic) with comprehensive implementations. Each metric can be instantiated with flexible configurations and applied to model outputs to generate per-unit scores. The framework handles statistical validation and supports both individual metric application and batch processing with the quantus.evaluate() function that aggregates results across multiple metrics and XAI methods.

Step 5 - ANALYZE The toolkit provides strong analytical capabilities through its high-level quantus.evaluate() function that enables large-scale evaluations across multiple XAI methods and metrics simultaneously. It supports aggregation functions (like np.mean) for statistical summaries, handles multiple data types (image, time-series, tabular), and includes extensive tutorials showing metric parameterization analysis, robustness analysis, and sensitivity studies. The framework enables comparative analysis between different explanation methods with quantitative evidence.

Medium-Coverage Phases
Step 1 - PLAN Quantus provides some planning support through its comprehensive metric categorization and extensive documentation. The toolkit offers clear guidance on metric selection across different evaluation criteria and includes tutorials for different use cases. However, it lacks automated configuration generation, formal task definition frameworks, or systematic evaluation planning tools. Users must manually select appropriate metrics and configure evaluation parameters based on domain knowledge.

Step 2 - PREPARE The library offers basic data preparation capabilities with support for multiple data modalities (image, time-series, tabular) and framework integration (PyTorch, TensorFlow). It includes helper functions for loading models and datasets, and provides built-in explanation method integration with popular frameworks (Captum, tf-explain, Zennit). However, it lacks comprehensive data preprocessing pipelines, automated dataset validation, or specialized preparation for different evaluation scenarios.

Step 3 - EXECUTE Quantus supports execution through its metric evaluation framework and built-in quantus.explain function that includes popular explanation methods. The library handles model inference for explanation generation and provides both pre-computed attribution support and on-the-fly explanation computation. However, it lacks comprehensive execution orchestration, advanced resource management, or specialized handling for different evaluation scenarios beyond XAI explanation evaluation.

Low-Coverage Phases
Step 6 - SHIP The library provides basic output generation through metric scores and supports result aggregation via quantus.evaluate(). However, it lacks comprehensive artifact management, standardized report generation, dashboard creation, or automated export capabilities. Results are primarily returned as numerical scores without extensive packaging for consumption or compliance reporting.

Step 7 - GOVERN Quantus offers minimal governance support with basic reproducibility through documented metric implementations and version control. The library includes testing frameworks and CI/CD integration capabilities, but lacks comprehensive compliance validation, automated quality gates, drift detection, or production monitoring capabilities. The disclaimer about metric implementation verification by original authors indicates limited validation governance.
