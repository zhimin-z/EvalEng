# S3C1: Output Format Validation

## Brief
The harness validates that model outputs conform to benchmark-specified format requirements before measurement, identifying format compliance failures separately from construct measurement failures.

## Essential Features

1. **Format Specification Validation**: Validate that model outputs conform to benchmark-specified formats (JSON schema, structured fields, regex patterns)
2. **Pre-Measurement Validation**: Check format compliance before applying scoring protocols to separate format from content issues
3. **Format vs. Content Error Separation**: Distinguish format compliance failures from construct measurement failures with separate error reporting
4. **Parsing & Field Extraction**: Parse outputs to extract required fields and validate structural requirements
5. **Format Violation Reporting**: Provide clear, actionable error messages indicating specific format violations and expected formats
