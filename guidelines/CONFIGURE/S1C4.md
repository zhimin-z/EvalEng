# S1C4: Baseline Specification & Configuration

## Brief
The harness enables configuration of baseline systems that establish comparative context, ensuring fair comparison conditions across all systems being evaluated.

## Essential Features

1. **Baseline System Specification**: Define baseline systems for comparison (random baseline, rule-based, human performance, previous models)
2. **Fair Comparison Configuration**: Ensure baselines are evaluated under identical conditions as target systems (same prompts, same protocols, same resources)
3. **Multiple Baseline Support**: Configure and evaluate multiple baseline types simultaneously for comprehensive comparative context
4. **Baseline Type Selection**: Support diverse baseline types (random guessing, majority class, human annotators, previous state-of-the-art)
5. **Evaluation Parity Validation**: Validate that baseline configurations maintain fair comparison conditions across all evaluated systems
