# S3C7: Fairness Assessment

## Brief
The harness evaluates whether performance metrics are consistent across demographic groups and subpopulations, identifying measurement disparities.

## Essential Features

1. **Subgroup Performance Computation**: Compute performance metrics separately for demographic groups and subpopulations
2. **Demographic Parity Testing**: Test whether outcomes are equally distributed across demographic groups
3. **Equalized Odds Testing**: Test whether true positive rates and false positive rates are equal across groups
4. **Predictive Parity Testing**: Test whether positive predictive value is consistent across groups
5. **Intersectional Analysis**: Analyze fairness across combinations of protected attributes (e.g., race × gender)
6. **Statistical Significance in Subgroups**: Test for statistically significant disparities between subgroups
7. **Rater Demographic Analysis**: Analyze rater biases and agreement patterns by demographic when using human judgment
8. **Bias Detection & Reporting**: Identify, quantify, and report fairness violations and measurement disparities


## Evidence to Look For
- Demographic subgroup tracking in data
- Subgroup performance computation code with per-group metric calculation
- Fairness criterion implementation for multiple fairness definitions
- Equalized odds computation with true positive and false positive rates
- Intersectional analysis code for multi-attribute combinations
- Statistical testing within subgroups with appropriate adjustments
- Sample size adjustment for smaller subgroups
- Rater demographic tracking if applicable
- Rater agreement analysis by demographic
- Rater-ratee concordance analysis
- Fairness violation detection and flagging
- Bias reporting with quantified disparities
- Test cases showing fairness analysis across multiple protected attributes
- Documentation of fairness criteria and methodology

## Evaluation Criteria

### Coverage Calculation

Grade mapping:
- Absent (0-30%): <2 features present
- Partial (30-80%): 2-4 features present
- Present (80-100%): 5+ features present

### Documentation Grading
- Minimal: Sparse or unclear documentation; users would struggle to understand how to use the features
- Moderate: Basic documentation exists (e.g., README examples, some API docs) but may lack detail or completeness
- Comprehensive: Thorough documentation with clear examples, API references, and explanations of all features
