# S2C6: Error Recovery & Resilience

## Brief
The harness implements automatic recovery mechanisms that enable evaluation to complete despite transient failures, distinguishing recoverable from unrecoverable errors.

## Essential Features

1. **Automatic Retry Logic**: Retry failed operations with configurable retry policies (exponential backoff, max attempts, timeout settings)
2. **Transient Error Detection**: Identify temporary failures (network errors, rate limits, timeouts) that can be recovered through retry
3. **Error Classification**: Distinguish recoverable errors (temporary) from unrecoverable errors (permanent failures requiring intervention)
4. **Graceful Degradation**: Continue evaluation when possible despite partial failures, skipping only unrecoverable instances
5. **Comprehensive Error Logging**: Maintain detailed logs of all errors, recovery attempts, and outcomes for debugging and analysis
