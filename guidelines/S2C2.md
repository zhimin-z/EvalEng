# S2C2: Schema Validation & Consistency

## Brief
The harness validates that input data conform to expected schemas before expensive operations, detecting data quality issues and consistency violations early.

## Essential Features

1. **Schema Conformance Validation**: Validate that data structure matches expected schemas before processing (field names, nesting structure)
2. **Early Validation Execution**: Perform validation checks before expensive operations (model inference, scoring) to fail fast
3. **Data Type Checking**: Verify that field types match expected types (strings, integers, lists, nested objects)
4. **Required Field Validation**: Ensure all required fields are present and non-null before proceeding
5. **Cross-Field Consistency Checking**: Identify consistency violations across related fields (e.g., answer count matches question count)
